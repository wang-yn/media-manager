# Emby-like 媒体库交互设计

## 状态

- 日期：2026-07-09
- 状态：设计已批准，等待实现计划
- 范围：调整 Media Manager 前端交互结构，让首屏只展示媒体库内容，并把所有设置功能移动到设置页

## 背景

当前 MVP 首屏同时展示系统状态、媒体目录管理和影视列表。这个结构更像管理后台，不符合本轮目标。

本轮目标是参考 Emby 的交互逻辑：用户进入产品后先看到媒体库内容；媒体库配置、服务状态、路径和其他管理能力进入设置或管理页面。

参考资料：

- Emby Library Setup: `https://emby.media/support/articles/Library-Setup.html`
- Emby Server Dashboard / 管理入口参考: `https://emby.media/support/articles/Connectivity.html`
- Emby Installation / 首次设置流程参考: `https://emby.media/support/articles/Installation.html`

## 目标

- 首屏只展示媒体库内容。
- 将添加媒体目录、目录列表、后端状态、配置路径、媒体根路径、TMDB key 状态等设置功能移动到设置页。
- 保留当前媒体条目的手工刮削、候选选择、重命名预览和执行能力。
- 不引入新的后端 API，不引入数据库或后台任务。
- 不引入 React Router；MVP 用 hash 或内部 state 实现页面切换。

## 非目标

- 不做海报墙、播放、用户账号、权限控制。
- 不做 Emby 完整复制，只参考“内容优先 + 设置分离”的交互逻辑。
- 不新增自动扫描、后台任务、批量刮削、字幕下载。
- 不新增 UI 框架或路由依赖。

## 信息架构

### 首页 `/`

首页只显示媒体库入口。

展示内容：

- 顶部导航：产品名、刷新按钮、设置入口。
- 媒体库网格：每个媒体库一张入口卡片。
- 空状态：没有媒体库时，显示“尚未配置媒体库”和“前往设置”按钮。

媒体库卡片字段：

- 媒体库名称。
- 类型：电影或剧集。
- 条目数。
- NFO 缺失数。
- 最近错误数。首版前端没有持久错误历史时显示为 `0`。

首页不展示：

- 添加媒体目录表单。
- 媒体根路径。
- 配置文件路径。
- 后端状态面板。
- TMDB key 提示。
- 单个影视条目的刮削或重命名按钮。

### 媒体库详情 `/libraries/:id`

媒体库详情只展示当前媒体库内的内容和内容操作。

展示内容：

- 顶部：返回媒体库、媒体库名称、刷新按钮、设置入口。
- 影视条目表格。
- 行级操作：刮削、候选选择、预览重命名、执行重命名。
- 行级状态：NFO 是否存在、路径、季/集信息、操作错误。

过滤规则：

- 前端复用 `GET /api/media` 的全部扫描结果。
- 通过 `library` 或 `library_path` 匹配当前媒体库。
- 不新增后端过滤接口。

媒体库详情不展示：

- 添加媒体目录表单。
- 全局系统状态。
- 配置路径和媒体根路径。

### 设置页 `/settings`

设置页承载所有管理功能。

展示内容：

- 系统状态：后端状态、配置文件路径、媒体根路径。
- TMDB key 状态：显示是否可用或缺失，不显示密钥值。
- 媒体目录管理：当前目录列表、新增目录表单、类型选择 `movie/series`、路径校验错误。
- 预留但首版不实现：整理模板、扫描策略、元数据源扩展。

设置页不展示：

- 影视条目的刮削按钮。
- 重命名预览和执行按钮。
- TMDB 候选列表。

## 导航设计

MVP 不引入 React Router。前端用 hash 或内部 state 表示三个视图：

- `home`
- `library`
- `settings`

建议 hash 形式：

- `#/`
- `#/libraries/<encoded-library-id>`
- `#/settings`

导航规则：

- 产品名点击回到首页。
- 首页媒体库卡片点击进入库详情。
- 设置入口从任意页面进入设置页。
- 库详情页提供返回媒体库入口。

## 数据流

页面初始化时加载：

- `GET /api/libraries`
- `GET /api/media`

只有进入设置页或刷新设置页时加载：

- `GET /api/health`

设置页新增目录：

- 调用 `POST /api/libraries`
- 成功后刷新 `libraries` 和 `media`
- 失败时在设置页显示后端结构化错误

库详情行级操作：

- 刮削候选：`POST /api/media/{id}/metadata/search`
- 应用元数据：`POST /api/media/{id}/metadata/apply`
- 重命名预览：`POST /api/media/{id}/rename/preview`
- 执行重命名：`POST /api/media/{id}/rename/apply`

## 错误呈现

- 首页：只显示媒体库加载失败这类阻断错误。
- 库详情：显示媒体条目操作错误，例如 TMDB 请求失败、NFO 写入失败、重命名冲突。
- 设置页：显示配置写入、目录校验、TMDB key 缺失、健康检查失败。

错误文案继续使用后端结构化错误：

- `message`
- `detail`
- `path`

前端不把详细错误替换成泛化文案。

## 视觉与交互约束

- 首页采用内容优先的媒体库网格，不做营销式 hero。
- 媒体库卡片是内容入口，不是设置卡片。
- 设置页可以更密集、更像管理面板。
- 桌面优先，窄屏下媒体库卡片单列，表格横向滚动。
- 所有按钮和表单控件保持键盘可访问。

## 实现边界

本次实现只调整前端视图结构和状态组织。

允许修改：

- `frontend/src/App.tsx`
- `frontend/src/style.css`
- 必要的前端类型和辅助函数

通常不需要修改：

- 后端 API
- TOML 配置格式
- Dockerfile
- README

如果实现中发现当前 API 无法支持媒体库卡片统计，可以在前端从 `libraries` 和 `media` 聚合，优先不扩后端。

## 测试与验证

必须验证：

- `cd frontend && npm run build`
- `PYTHONPATH=backend/src .venv/bin/python -m unittest discover backend/tests`

手动烟测：

- 打开首页，只能看到媒体库内容和顶部导航，不出现设置表单或系统状态面板。
- 点击媒体库卡片进入库详情，只看到该库内条目。
- 设置页能查看系统状态并添加媒体目录。
- 库详情页仍能触发现有刮削和重命名操作。
