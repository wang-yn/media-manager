# 媒体库质量治理只读检查设计

## 状态

- 日期：2026-07-15
- 状态：设计已批准，等待实现计划
- 范围：为媒体库增加同步、只读的目录结构质量检查，帮助用户发现空目录、无视频目录、孤立旁路文件、电影结构异常、剧集结构异常和小视频文件。

## 背景

当前项目已经支持媒体库扫描、元数据刮削、字幕下载、自动重命名、删除、目录大小和详细文件查看。用户仍需要一种入口，快速发现媒体库里不符合整理规范的目录和文件。

本轮不做自动修复，也不把异常目录伪装成媒体条目。检查结果作为独立的质量治理结果返回，由用户后续手工处理。

## 目标

- 提供 `GET /api/audit` 同步接口，按媒体库分组返回检查结果。
- 前端在媒体库页面提供“检查媒体库”入口。
- 使用弹窗展示检查结果、问题总数、类型筛选、问题表格、重新检查和 CSV 导出。
- 结果只展示媒体库内相对路径，不展示宿主机绝对路径。
- 小视频文件检查阈值固定为 `100 MiB`。
- 单个目录读取失败时返回为问题行，不中断整个检查。
- 继续忽略 `.@__thumb` 目录。

## 非目标

- 不做后台任务、自动扫描或定时检查。
- 不持久化检查结果。
- 不自动删除、移动、重命名或修复文件。
- 不做重复文件检测。
- 不引入数据库、哈希计算、任务队列或新依赖。
- 不修改现有媒体扫描结果结构，把异常目录混入 `GET /api/media`。

## 整体方案

新增一个独立的同步检查接口：

```text
GET /api/audit
```

后端遍历已配置媒体库，复用现有视频扩展名、忽略目录和相对路径规则，按媒体库返回质量问题。接口只读，不创建、删除或移动任何文件。

前端点击“检查媒体库”后调用接口，并把本次结果保存在当前页面会话内。用户关闭弹窗、刷新页面或重新检查后，旧结果直接丢弃。

该方案比后台任务更小，也符合当前项目“手工同步操作”的第一版边界。

## 问题类型

### `empty_directory`

目录内没有任何有效文件或有效子目录。

忽略 `.@__thumb` 后，如果目录为空，则返回该问题。媒体库根目录本身不作为空目录问题返回。

### `directory_without_video`

目录内存在文件，但目录树内没有任何受支持的视频文件。

典型情况包括只包含图片、说明文件、压缩包或其他无关文件的目录。

### `orphaned_sidecar`

存在旁路文件，但同目录下没有可对应的视频文件。

首版检查以下扩展名：

- `.nfo`
- `.srt`
- `.ass`
- `.ssa`

### `invalid_movie_layout`

电影媒体库中发现不符合电影目录规范的结构。

规范要求：

- 每部电影必须有自己专属的文件夹。
- 文件夹名和视频文件名都应同时包含中文名和英文名，并包含圆括号年份，例如 `电影名 Movie Name (2024)`。
- 视频文件不能直接放在电影媒体库根目录下。

首版只检测可稳定判断的问题：

- 视频文件直接位于电影媒体库根目录。
- 一个电影目录内存在多个顶层视频文件时，标记为结构风险。
- 文件夹名或视频文件名缺少年份圆括号，例如 `(2024)`。
- 文件夹名或视频文件名缺少中文字符或英文字母。

### `invalid_series_layout`

剧集媒体库中发现不符合剧集目录规范的结构。

规范要求：

- 三层结构：剧集文件夹 -> 季度文件夹 -> 单集文件。
- 季度目录使用 `Season xx`。
- 单集文件名必须包含 `SxxExx`。

首版检测：

- 视频文件直接位于剧集媒体库根目录。
- 视频文件直接位于剧集文件夹下，缺少季度目录。
- 季度目录命名不符合 `Season xx`。
- 单集文件名缺少 `SxxExx`。

### `small_video_file`

受支持的视频文件小于 `100 MiB`，即 `104,857,600` 字节。

结果中额外返回实际文件大小，前端显示为 MiB。该问题只提示风险，不判断文件一定损坏。

### `read_error`

遍历目录或读取文件状态失败。

该问题用于保留错误信息，例如权限不足、路径不存在、文件系统异常。单个读取失败不让整个接口失败。

## 返回数据

接口返回按媒体库分组的数据：

```json
{
  "libraries": [
    {
      "name": "电影",
      "type": "movie",
      "issues": [
        {
          "id": "movie:invalid_movie_layout:Bad Movie/file.mkv",
          "type": "invalid_movie_layout",
          "message": "电影文件直接位于媒体库根目录",
          "library": "电影",
          "relative_path": "Bad Movie.mkv",
          "size_bytes": null,
          "detail": null
        }
      ]
    }
  ]
}
```

字段说明：

- `id`：稳定 ID，由媒体库、问题类型和相对路径派生。
- `type`：问题类型。
- `message`：中文原因。
- `library`：媒体库名称。
- `relative_path`：媒体库内相对路径。
- `size_bytes`：仅小视频问题使用。
- `detail`：读取失败或结构风险的补充说明。

接口响应不返回宿主机绝对路径。

## 前端交互

媒体库页面提供“检查媒体库”按钮。

点击后：

- 按钮进入加载状态。
- 调用 `GET /api/audit`。
- 成功后打开结果弹窗。
- 失败时在页面错误区域显示后端结构化错误。

结果弹窗包含：

- 标题：媒体库检查结果。
- 问题总数。
- 按类型筛选的标签或按钮。
- 问题表格：问题类型、相对路径、详情。
- `重新检查` 按钮。
- `导出 CSV` 按钮。
- 右上角 `X` 关闭按钮。

无问题时显示明确空状态：“未发现结构问题”。

筛选只影响弹窗内展示，不重新请求后端。重新检查会再次请求接口并覆盖当前结果。

## CSV 导出

CSV 在前端基于当前检查结果生成，不新增后端导出接口。

字段：

- 媒体库
- 类型
- 问题
- 相对路径
- 详情
- 大小

导出全部检查结果，不受当前筛选影响。

## 错误处理

- 配置文件无法读取、媒体库配置无效、媒体库根路径不可访问时，接口返回 HTTP 错误。
- 单个子目录无法读取时，接口继续检查其他路径，并返回 `read_error` 问题行。
- 前端保留后端返回的 `message`、`detail` 和路径信息，但只显示相对路径。
- CSV 导出失败时仅在弹窗内显示错误，不影响检查结果。

## 文件安全

- 检查接口只调用文件遍历和状态读取。
- 不创建 NFO、字幕、目录或临时文件。
- 不删除、移动或重命名任何文件。
- 所有返回路径必须限制在对应媒体库根目录内。

## 实现边界

预计修改：

- `backend/src/media_manager/media.py`：增加质量检查函数、问题类型和大小格式所需数据。
- `backend/src/media_manager/server.py`：增加 `GET /api/audit`。
- `backend/tests/test_media.py` 或新增相邻测试：覆盖检查规则。
- `frontend/src/App.tsx`：增加检查入口、结果弹窗、筛选和 CSV 导出。
- `frontend/src/style.css`：补充弹窗表格和标签样式。

不新增后端模块、前端状态管理库或 UI 组件库。优先复用现有媒体库、扫描、弹窗、错误显示和按钮样式。
